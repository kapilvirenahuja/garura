01
Market Context

Problem

Engineering teams at mid-size SaaS companies (50-500 engineers) lack real-time visibility into how code changes impact production system health. Existing APM tools surface symptoms (latency spikes, error rates) but cannot correlate them back to specific deployments, pull requests, or code authors within minutes of a release. Teams spend an average of 47 minutes per incident on root-cause attribution, and 30% of rollbacks happen reactively after customer complaints rather than proactive detection.

Target Users

Platform Engineering Lead
Goal
Reduce MTTA from 47 minutes to under 5 minutes for deployment-correlated incidents
Frustration
Current tools require manually cross-referencing deploy logs, APM dashboards, and git history across 3-4 systems
Context
Manages platform for 80-200 engineers, oversees 15-40 deploys/day
On-Call Engineer
Goal
Immediately identify which deployment caused a production anomaly when paged
Frustration
Gets paged at 3am with a latency spike but has no way to correlate it to the deploy that shipped 12 minutes earlier
Context
Rotates on-call weekly, handles 2-5 pages per shift
VP of Engineering
Goal
Quantitative view of deployment risk across teams for data-driven release decisions
Frustration
Cannot answer "which team ships the riskiest deploys?" without a multi-week analysis project
Context
Oversees 5-12 teams, reports to CTO and board

Differentiators

Automatic Deploy-to-Impact Correlation
Ingests deploy events from CI/CD and correlates with APM anomalies in real-time using causal inference. Works within 60 seconds.
Change-Aware Anomaly Detection
Baselines adjust for known deploys — reducing false-positive pages by up to 60%.
PR-Level Attribution
Attributes impact to specific pull requests, not just the deployment.
Deployment Risk Score
Pre-deploy risk scoring based on historical patterns, blast radius, and code complexity delta.

Risks

APM Vendor Lock-in Dependency
If APM vendors build equivalent features natively, they commoditize our core value. Mitigation: proprietary causal inference beyond timestamp matching.
Data Volume Scaling Costs
Real-time correlation generates significant processing volume at scale. Mitigation: tiered processing with sampling for low-risk deploys.
False Attribution Reputation Risk
Incorrect attribution erodes trust quickly. Mitigation: confidence scoring, "uncertain" state, and feedback loops.

03
Scope

Assumptions

Target customers already use a supported CI/CD system and APM tool
Deploy events capturable via webhook/API without pipeline changes
APM metric data accessible via vendor APIs with 1-minute resolution
Teams deploy at least 5 times per week
Budget authority for $5K-$12K/month tooling at the mid-market

Out of Scope

Infrastructure Monitoring
Rationale
Pulse is a correlation layer between CI/CD and APM, not an APM replacement
CI/CD Pipeline Execution
Rationale
We ingest deploy events but do not execute deployments
On-Premises Infrastructure
Rationale
Cloud-native SaaS only in V1; on-prem evaluated post-PMF
Mobile Application Monitoring
Rationale
Mobile release cycles have fundamentally different deploy-to-impact timelines
