# discover-product

Given a product idea or problem statement, discover the market opportunity, draft a product vision with strategic goals and market positioning, validate the vision for completeness, and lock it when ready. The recipe operates in three phases (DRAFT, VALIDATE, LOCK) invoked independently. DRAFT produces a vision and an interactive HTML brief for human review at a checkpoint. VALIDATE checks vision completeness with cycle-back support. LOCK finalizes the vision for downstream planning.

Owner: recipe
Depends on: Step 3

### Phase: Checkpoint

**Step 2 — Validation Review**
Owner: recipe
Depends on: Step 1

Present validation summary: completeness_score, checklist table, issues by severity.

**If `ready_for_lock: true`:**
- Present: "Vision is ready to lock. Run `/discover-product --phase lock --artifact {path}` when ready."
- `Tether` -> write VALIDATE_PASSED checkpoint, proceed to Step 3
- `Vanish` -> halt

**If `ready_for_lock: false`:**
- Present issues. `Tether` to proceed with risks, `Vanish` to cycle back.
- On `Vanish` (cycle-back, C9):
  - Read iteration count from `{product_base}/{slug}/status/discover-product.json`
  - If iterations < 2: re-invoke `product-strategist` with `draft-product-vision` + feedback from validation issues. Increment iteration count. Return to Step 1.
  - If iterations >= 2: halt with F5 — "Maximum cycle-back iterations reached."
- On `Tether` with blockers: note risk in checkpoint, write APPROVED checkpoint.

## Scenario Validation

Run after the active phase's workflow completes, before evidence.

**DRAFT scenarios:**
- **SCE-1 (S1):** HTML brief contains market opportunity, vision with strategic goals, business analysis, and feedback panel with per-section toggles and freeform textarea. Founder/PM can evaluate and provide structured feedback.

**VALIDATE scenarios:**
- **SCE-3 (S3):** Validation result contains completeness_score (0-100), ready_for_lock (boolean), 5-field checklist, and severity-classified issues. PM can determine lock-readiness.

**LOCK scenarios:**
- **SCE-2 (S2):** Locked vision has >=3 strategic goals, >=2 target user personas with roles, quantifiable success metrics, and named competitors. Engineering Lead can derive milestones.

## Recovery

When an agent returns structured failure:
1. Read `domain_assessment.responsible_domain`
2. Invoke responsible agent with fix context + original intent
3. Max 2 retries per step. After that, halt with full failure context.

```json
    "draft-checkpoint": { "status": "pending" },
    "draft-evidence": { "status": "pending" },
    "validate-vision": { "status": "pending" },
    "validate-checkpoint": { "status": "pending" },
    "validate-evidence": { "status": "pending" },
    "lock-vision": { "status": "pending" },
    "lock-evidence": { "status": "pending" }
  }
}
```

**Executor loop:**
```
determine slug from input (intent text or artifact path)
check for status file at {product_base}/{slug}/status/discover-product.json

for each step in active phase's compiled order:
  if status file shows step "completed" -> skip
  if status file shows step "in_progress" -> reset to pending
  mark step "in_progress" in status file
  execute step
  mark step "completed" in status file
```

**Fresh start:** No status file -> execute all steps, create on first step.
**Resume:** Status file exists -> skip completed, continue from first incomplete in active phase.
**Cross-phase:** VALIDATE and LOCK phases read but do not reset DRAFT task statuses. Each phase manages only its own tasks.
