# capture-learning

Given a completed issue number, archive the issue's STM directory to `_archive/{YYYY-MM}/{issue}/` to keep the active workspace clean, preserving the full directory structure and all files so the issue's history remains accessible for audit. The issue must be in a closed state before archival proceeds.

**Executor loop:**
```
parse issue number from input
check for status file at {stm_base}/{issue}/status/capture-learning.json

for each step in compiled order:
  if status file shows step "completed" -> skip
  if status file shows step "in_progress" -> reset to pending (may not have finished)
  mark step "in_progress" in status file

```

**Fresh start:** No status file exists -> execute all steps, create status file on first step.
**Resume:** Status file exists -> skip completed, continue from first incomplete.
**Pause:** User terminates session. Status file reflects last completed step. Next invocation resumes.

**Note on archival and status file:** The status file lives at `{stm_base}/{issue}/`. Step 2 moves this directory to archive. If resuming after Step 2 completed, check the archive location for the status file: `{stm_archive}/{YYYY-MM}/{issue}/status/capture-learning.json`.
