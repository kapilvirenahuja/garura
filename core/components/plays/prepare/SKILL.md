# prepare

Given an epic, produce a complete set of locked, implementation-ready design artifacts. Upstream locked artifacts from specify (epics, scope, quality profile), design (design-spec, screens), and arch (5 architecture artifacts) are REQUIRED inputs — hard halt if absent. Before generating any design, the play deeply understands the codebase through architecture inference (reading locked architecture as input), test surface mapping, dependency graph construction, and git history analysis — all in parallel. It then computes the blast radius via test impact analysis, identifies coverage gaps, and specifies baseline tests for uncharacterized code paths. Only after the blast radius is fully characterized and the user approves does LLD generation begin. The LLD (tech.yaml) defines interface-level design: API contracts, internal interfaces, frontend specs, backend specs, data specs, communication channels, and mock strategy — NOT file-level before/after code. Scenarios.yaml provides 3-tier verification scenarios. Plan.yaml provides the task DAG. All artifacts locked after human review. All work is issue-mapped. All intermediate data is stored on disk in STM.

|----|-----|-------------------|-----------|---------|
