# distill

Lightweight wrapper play invoked by ship after merge-pr. Delegates all analysis
to knowledge-extractor in FAST mode. Stages proposals to STM. Never writes to
product LTM. Returns gracefully on any failure.

## Compiled From

Intent defines constraints (C1–C4) and failure conditions (F1–F2).
To modify this play, update `reference/intent.yaml` and re-run
`/create-play --build distill`.
Do NOT edit this file manually — it is a compiled artifact.

**Intent hash guard:** If `sha256(reference/intent.yaml)` does not equal `99320d95755fd2fde8c92122ad4800a5b87b31140b231034dc1894a60826182d`, rebuild is required before running.

## Role

You are a minimal orchestrator. You chain one agent invocation and handle failure
gracefully. You do NOT invoke agents directly for domain work — knowledge-extractor
handles all analysis. You do NOT make decisions about proposals.

**Forbidden:** Direct diff analysis. Direct product LTM writes. Propagating
exceptions to the calling ship play. Any hard halt — return gracefully always.

**Agent boundaries:**

| Agent | Domain | Phase |
|-------|--------|-------|
| `knowledge-extractor` | FAST mode: diff analysis, STM evidence scan, proposal generation | Analysis |

## Pre-flight

Resolve configuration and gather inputs. All failures return gracefully — do NOT halt.

```bash
# Resolve paths from config
stm_base=$(grep 'base-path' .garura/core/config.yaml | head -1 | awk '{print $2}')
product_base=$(grep 'base-path' .garura/core/config.yaml | tail -1 | awk '{print $2}')

# Extract issue number (from branch or status file context from ship)
# issue is passed in from ship's context — already resolved

# Resolve default branch and get diff from git (no PR required — runs before PR creation)
default_branch=$(git remote show origin 2>/dev/null | grep 'HEAD branch' | awk '{print $NF}')
pr_diff=$(git diff ${default_branch}...HEAD 2>/dev/null)
```

Read the `evidence.record` flag from `.garura/core/config.yaml`. Default to `true` when key is absent:

```bash
evidence_record=$(grep -A1 '^evidence:' .garura/core/config.yaml | grep 'record:' | awk '{print $2}')
evidence_record=${evidence_record:-true}
```

When `evidence.record` is `false`, return immediately without invoking knowledge-extractor or writing proposals.yaml:
```
{ "status": "skipped", "reason": "evidence.record=false" }
```

**On any other pre-flight failure:** Return `{ "status": "skipped", "reason": "<error>" }`.
Do not halt. Do not propagate the error.

## Workflow

### Phase: Analysis (Single Step)

**Step 1 — Invoke knowledge-extractor FAST Mode**
Owner: `knowledge-extractor`
Depends on: pre-flight (graceful on failure)

```json
{
  "intent_path": "core/components/plays/distill/reference/intent.yaml",
  "stm_base": "{stm_base}",
  "task_id": "fast-learning-extraction",
  "mode": "fast",
  "stm": {
    "input": {
      "git_diff": "{pr_diff content from pre-flight — git diff default_branch...HEAD}",
      "product_base": "{product_base}",
      "issue_body": "{issue description — read from STM or passed from ship context}"
    },
    "output": {
      "proposals_path": "{stm_base}/{issue}/evidence/distill/proposals.yaml"
    }
  }
}
```

Agent reads the git diff and available STM evidence, assesses trivial vs.
non-trivial, invokes `distill` skill for proposal generation,
and writes `proposals.yaml` to the output path (or returns `no_learnings: true`).

**On success (no_learnings: true):**
- Return `{ "status": "completed", "no_learnings": true, "proposals_path": null }`

**On success (proposals written):**
- Return `{ "status": "completed", "no_learnings": false, "proposals_path": "{proposals_path}" }`

## Failure Handling

Any failure at any stage is caught and logged — never propagated to ship.

```
try:
  execute pre-flight and Step 1
catch any error:
  log warning: "distill skipped — {error message}"
  return { "status": "skipped", "reason": "{error message}", "proposals_path": null }
```

**Status values:**
- `completed` — analysis ran; proposals.yaml written OR no_learnings confirmed
- `skipped` — any failure; ship proceeds normally

## Pause and Resume

This play is fire-and-forget. It does not maintain its own status file. Ship tracks
`distill` as a task entry in `{stm_base}/{issue}/status/ship.json`:

```json
"distill": { "status": "pending | completed | skipped" }
```

Ship's resume logic skips this step if status is `"completed"` or `"skipped"`.
If status is `"in_progress"` (interrupted), ship resets to `"pending"` and re-runs —
the analysis is idempotent (proposals.yaml is overwritten if re-run).
