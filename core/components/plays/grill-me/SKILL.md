# grill-me

A scoping discipline that grills a human about a proposed feature, epic, technical
decision, or user-experience design decision by **pushing back** with cited
specificity until the shape is concrete enough that downstream plays cannot drift.
At session close it writes the locked shape and every implied target-shape change
as one atomic bundle, files one code follow-up issue, and exits. It never touches
code; the bundle moves only the target shape documented in product LTM.

### Phase: Execution (round loop)

**Step 3 — Round loop until termination**
Owner: play · Depends on: Step 2 · `Task: TaskUpdate [T3] → in_progress`

Initialize session state at `{stm_base}/{issue}/evidence/grill-me/session-state.yaml`
with anchor_lock_path, register, empty rounds[], empty accepted_gaps[], empty
code_implications[], empty proposed_epic.

Loop until termination (C9: `defensible` OR `human_good_enough`). For each round:

1. **Mode reminder (C10).** Before asking any question, write a stamp
   `mode_reminder_recorded: true` into the round's session-state entry and
   restate to yourself that this is grill mode — push back, do not agree.
2. **TaskCreate** `[T3.R{n}]` for the round with `addBlockedBy` on the prior
   round task (or on T2 if R1).
3. **Draft 3-4 gap-closing questions** in the register from the anchor lock —
   `product` for epic/feature anchors, `technical` for tech/design anchors.
   No machine/agent jargon. No A/B/C menus. Ask in plain prose.
4. **Receive answers from the human.** Record verbatim in
   `rounds[].answers`.
5. **Synthesise** a plain-language statement of what the shape looks like
   after these answers. Write to `rounds[].synthesis_text`.
6. **Invoke `check-grill-tensions`** via Skill tool:

   ```json
   {
     "anchor_lock_path": "{stm_base}{issue}/evidence/grill-me/anchor-lock.yaml",
     "round_synthesis_text": "{synthesis text from step 5}",
     "round_id": "R{n}",
     "prior_tensions_resolved": [...],
     "output_path": "{stm_base}{issue}/evidence/grill-me/rounds/R{n}-tensions.yaml"
   }
   ```

   Parse the returned `tension_count`. Record `tensions_report_path` in
   `rounds[]`.

7. **If `tension_count > 0`:** for each tension, issue exactly ONE plain-language
   push-back to the human (C2) that quotes the tension's `cited_text` from the
   skill output verbatim (C1) and names the contradiction specifically. Receive
   the re-answer.
   - If the re-answer resolves the tension: record `resolution_text` and
     `applies_to[]` (which target-shape artefacts this resolution implies an
     edit to — sourced from the touchpoints inventory). Mark the tension
     resolved.
   - If the re-answer does not resolve: record the gap, advance — do NOT
     re-push the same weak answer in this round (C2/F2).
   - If the human signals disengagement (asks to stop, signals fatigue, stops
     answering): set `disengagement_at_round: R{n}`, move all live tensions to
     `accepted_gaps`, set `termination_reason: human_good_enough`, exit the
     loop.
8. **If `tension_count == 0`:** the synthesis is consistent — advance to the
   next round with a fresh gap-closing batch, OR if every defended item has
   been covered with no live tensions, set
   `termination_reason: defensible` and exit the loop.
9. **TaskUpdate** `[T3.R{n}] → completed`.

After exit, ensure session-state has:
- `termination_reason` in {`defensible`, `human_good_enough`}
- every live tension is either resolved or in `accepted_gaps`
- every resolution carries `resolution_text` AND either `applies_to[]` or a
  `documentation_only_in_shape_doc` tag

Before T3 completes, the orchestrator asks the human two final plain-language
questions (these populate the close bundle):
- "Are there code changes implied by anything we just agreed?" — capture any
  implications into `code_implications[]`.
- "What's the right name and current status for the epic that tracks this
  journey from today to the new target?" — capture into `proposed_epic`
  with `mode` (create/update), `epic_target` (if update), `proposed_name`
  (if create), `proposed_status` from the canonical 5 vocabulary, and a
  one-paragraph rationale.

**Step Evals (after this step):**
- **SE-1 (F1/C1):** every tension entry in each rounds/R*-tensions.yaml has a non-empty defended_id AND a non-empty cited_text
- **SE-2 (F2/C2):** for every round in session-state.yaml, push_back_count_by_weak_answer has no value greater than 1; every weak_answer carried past one round appears in accepted_gaps or in a resolution
- **SE-3 (F3/C3):** round transcript surfaces in session-state.yaml contain no jargon tokens, no A/B/C menus, and register matches the anchor lock's register field
- **SE-4 (F4/C4):** every entry in rounds[] has a non-empty synthesis_text AND a non-null tensions_report_path that resolves to an existing file
- **SE-6 (F6):** every tension marked resolved has a non-empty resolution_text AND either a non-empty applies_to list OR a documentation_only_in_shape_doc flag
- **SE-8 (F8/C9):** session-state.yaml termination_reason has value in {defensible, human_good_enough}
- **SE-9 (F9):** if disengagement_at_round is set, every tension live at that round appears in accepted_gaps AND no rounds with new push-backs exist after disengagement_at_round
- **SE-15 (C10):** every entry in rounds[] has mode_reminder_recorded=true

`Task: TaskUpdate [T3] → completed`

## Recovery

Per-failure recovery handoffs from expectation.yaml. Routing: `autonomous` =
play can self-correct deterministically; `human` = surface to the user.

- **REC1 (F1):** every push-back must carry an explicit citation — a constraint id or a quoted phrase from the target — before it reaches the human; uncited push-backs must be blocked at emission and rewritten to cite or dropped. (autonomous)
- **REC2 (F2):** a weak answer gets at most one push-back per round; on the second weak response the play must record the gap, advance, and only revisit the weakness in a later round if a new tension makes it material. (autonomous)
- **REC3 (F3):** rewrite the surface in plain product or technical prose matched to the anchor before showing the human; never expose schema fields, lettered menus, or framework internals. (autonomous)
- **REC4 (F4):** every round must end with both a synthesis and a tension check against declared constraints/rules/failure scenarios; a malformed round must be completed (synthesis + tension check appended) before any new round begins. (autonomous)
- **REC5 (F5):** do not enter round one against a target with nothing to defend against; halt with a plain-language explanation and direct the human to the upstream play that produces declarations. (human)
- **REC6 (F6):** a tension is resolved only when the final shape passes its constraint's defensibility check; if the human refuses to resolve, record it as an accepted gap, not as agreement. (human)
- **REC7 (F7):** the close bundle is approved as one unit; no part of it may be written before that single approval; on approval the bundle writes atomically (REC12). (human)
- **REC8 (F8):** terminate only on defensibility or explicit human good-enough; round counts, clocks, and token budgets are not termination triggers and must not be wired as such. (autonomous)
- **REC9 (F9):** on disengagement, pause immediately, record every unresolved tension into the shape document as an accepted gap, and exit cleanly — do not push back further. (human)
- **REC10 (F10):** the locked shape document is the play's deliverable; if it would not be written at end of session, write it from the in-memory session state before exit. (autonomous)
- **REC11 (F11):** before round one, lock the anchor to exactly one of the four permitted kinds via a single plain-language question; if the human cannot name one or names something outside the four, halt before round one; the locked anchor decides the constraints source and the C3 register for the whole session. (autonomous)
- **REC12 (F12):** the bundle writes as one transaction through the repo writer; stage every artefact, write all or none, and on any write failure roll back every staged change before exiting; the play exits in a state where either every artefact in the bundle is updated or none is. (autonomous)
- **REC13 (F13):** the writer rejects any path matching the project's code paths and the close bundle files exactly one code follow-up issue per session, whose body is the locked shape's "Code follow-up" section verbatim (empty case files with explicit "no code changes required" body, per S10). (autonomous)
- **REC14 (F14):** every close bundle includes an epic entry — created at status "planned" when grilling spawned a new epic or addressed a free-text idea or a sibling-split, refreshed in place when grilling refined an in-flight epic; the status field is set from the canonical 5-point vocabulary based on where code work currently sits. (autonomous)

## Pause and Resume

Status file at `{stm_base}/{issue}/status/grill-me.json`. Write after every step
completion:

```json
{
  "run_id": "grill-me-{ts}",
  "issue": "{issue}",
  "anchor_kind": "{kind}",
  "anchor_target_path": "{path}",
  "started_at": "{ISO 8601}",
  "current_step": "T3.R{n} | T4 | T5 | T6 | T7",
  "completed_steps": ["T1", "T2", "T3.R1", ...],
  "termination_reason": "{defensible | human_good_enough | pending}",
  "bundle_approval": "{pending | approved | vanish}",
  "commit_sha": "{sha or null}"
}
```

On resume:
1. `TaskList` to see state.
2. Skip completed tasks; reset any `in_progress` to `pending`.
3. Load session-state.yaml to recover round history and resolutions.
4. Continue from the first non-completed task. Round-loop sub-tasks pick up
   from the last completed round.

No re-planning. The compiled steps are the execution state.
