# reap

Post-epic learning extraction. After validate completes, read the build trinity and produce tiered, taxonomy-classified learning proposals staged in STM. Each proposal carries a two-level learning taxonomy (learning_category + sub_category) aligned to the core/components/memory/ KB structure. A human reviews proposals and Tethers to commit evidence or Vanishes to halt. No product LTM is written during this play.

Agent invokes `diff-context-baseline` → produces `context-diff.yaml` with taxonomy-classified findings.
Agent then invokes `draft-enrichment-proposals` → produces `proposals.yaml` at `{stm_base}/{issue}/evidence/reap/proposals.yaml`.
Agent also produces `adr-drafts/*.md` under `{stm_base}/{issue}/evidence/reap/adr-drafts/` for Tier 1 findings.

**Step 2 Eval — SE-1 (C7, F2):**
- Every proposal in `proposals.yaml` carries a non-empty `learning_category` field.
- `learning_category` value is one of `arch | domain | product | quality | standards` OR `learning_category_proposed: true` is set.

**Step 2 Eval — SE-2 (C7, F7, F8):**
- For proposals with `learning_category` in `{arch, quality, standards}`: `sub_category` is non-null and non-empty.
- For proposals with `learning_category` in `{domain, product}`: `sub_category` is null or empty.

**Step 2 Eval — SE-3 (C7a, F3):**
- For any proposal where `learning_category_proposed: true` or `sub_category_proposed: true`: `taxonomy_justification` block is present with non-empty `evidence_path`, `excerpt`, and `reasoning`.

**Step 2 Eval — SE-4 (C4, F4):**
- `proposals.yaml` output path is under `{stm_base}/{issue}/evidence/reap/`. No path under `{product_base}` or `~/.garura/core/memory/` appears in the proposals output.

**Step 2 Eval — SE-5 (C6, F5):**
- Every proposal with `tier: 1` carries an `adr_draft_path` and an `impact` block with at least `risk` and `downstream_artifacts`.

**Step 2 Eval — SE-6 (C8, F6):**
- When `spec-correction-manifest.yaml` was present at input, proposals sourced from check-drift carry `source: check-drift` and are not re-derived findings.

**Step 2 Eval — SE-7 (C11):**
- If no findings detected across all tiers, `proposals.yaml` is written with `total_proposals: 0` and an empty `proposals` list. Play does not treat this as an error.

```

```
