# prepare-implementation

Given an issue or epic, produce a complete set of locked, implementation-ready design artifacts. Before generating any design, the play deeply understands the codebase through architecture inference, test surface mapping, dependency graph construction, and git history analysis — all in parallel. It then computes the blast radius via test impact analysis, identifies coverage gaps, and specifies baseline tests for uncharacterized code paths. Only after the blast radius is fully characterized and the user approves does artifact generation begin. All work is issue-mapped. All intermediate data is stored on disk in STM.

| Resolve `product_base` from `core/config.yaml` | — | Hard halt — config required |
| Resolve issue number from branch name OR STM directory | C14 | Continue to epic check |
| If no issue: check for `--epic` flag AND `{product_base}/roadmap/roadmap.yaml` | C14 | Hard halt if neither issue nor epic (F19) |

```bash
# Resolve paths from config
stm_base=$(grep 'base-path' core/config.yaml | head -1 | awk '{print $2}')
product_base=$(grep 'base-path' core/config.yaml | tail -1 | awk '{print $2}')

# Parse issue from branch name (feat/183-... → 183)

# If no issue from branch, check for STM directory or caller-provided issue
if [ -z "$issue" ]; then
  issue="${INPUT_ISSUE:-}"
fi

# Halt condition (F19): no issue and no epic
if [ -z "$issue" ] && [ -z "${INPUT_EPIC:-}" ]; then
  echo "HALT: No issue resolved from branch and no --epic provided. Nothing to scope the work against. (F19)"
  exit 1
fi
```
