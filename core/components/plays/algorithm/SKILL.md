# algorithm

Optional play that reads tech.yaml, detects algorithmically complex `internal_interfaces` entries (via 11 indicator patterns or an explicit `algorithm_spec_required: true` flag), dispatches `tech-architect` to produce `reference-algorithms.md`, and gates the result on human approval before locking. CRUD epics with no qualifying interfaces exit with a skip notice ‚Äî no agent dispatch occurs.

Create ALL tasks via `TaskCreate` immediately after resolving `stm_base` ‚Äî before any domain work begins. The play owns this DAG. Agents MUST NOT update play-level tasks (they may `TaskCreate` discovered sub-work with `addBlockedBy`, but play-level tasks belong to the play).

### Phase: Execution

**Step 1 ‚Äî Draft Reference Algorithms**
Owner: `tech-architect`
Depends on: pre-flight (qualifying_interfaces is non-empty)
Task: `TaskUpdate [T1] ‚Üí in_progress` before dispatch. `TaskUpdate [T1] ‚Üí completed` after eval passes.

**Recovery:** If agent returns error, retry with fix context (max 2 attempts). After 2 failures, halt ‚Äî F5: report failing `interface_ids`.

**Step 1 Evals:**
- **SE-2 (F2):** `tech.yaml` is present and non-empty at the expected path before skill dispatch. If absent, the play halts with the message "prepare must run before algorithm" and the skill is never invoked.

- **SE-6 (F6):** `reference-algorithms.md` exists at `{stm_base}/{issue}/context/design/reference-algorithms.md` after skill execution. The file is non-empty.
- **SE-8 (C5):** The content and last-modified timestamp of `tech.yaml` is identical before and after skill execution. No other `prepare` artifact has been modified.
- **SE-9 (C6):** `reference-algorithms.md` exists at `{stm_base}/{issue}/context/design/reference-algorithms.md` and no other output file has been written outside this path.
- **SE-10 (C9):** For every `interface_id` in the skill's input list: a `## {interface_id}` heading exists in `reference-algorithms.md`, the section contains a complexity classification field, a pseudocode block delimited by a code fence using uppercase keywords only, an Invariants sub-section with at least one entry, and an Edge Cases sub-section with at least one entry.
- **SE-12 (C16):** `reference-algorithms.md` contains `status: DRAFT` in its header section. The approval gate has not been bypassed and no `status: LOCKED` marker is present before user approval.

## Recovery

Sourced from `reference/expectation.yaml` `recovery` ‚Äî one entry per failure condition. When the validator detects a tripped failure, it builds a recovery handoff plan from the matching entry and routes it per `handoff`: `autonomous` loops the fix back to the builder; `human` escalates for a manual call.

| ID | For | Symptom (trigger) | Direction | Handoff |
|----|-----|-------------------|-----------|---------|
| REC1 | F1 | `stm_base` cannot be resolved from `.garura/core/config.yaml` (key missing/empty or file unreadable) | `stm.base-path` needs a user fix ‚Äî the builder cannot author or repair the project config it merely reads | human |
| REC2 | F2 | `tech.yaml` absent at `{stm_base}/{issue}/context/design/tech.yaml` ‚Äî prepare context package not produced | prepare must run first to produce `tech.yaml` ‚Äî the builder cannot synthesize an upstream prepare artifact | human |
| REC3 | F3 | `tech.yaml` present but empty or unparseable YAML | re-run prepare or have its owner repair the file ‚Äî the builder cannot author the upstream design | human |
| REC4 | F4 | no complexity indicators and no `algorithm_spec_required: true`, yet the play did not exit gracefully | emit the skip notice and exit 0 with no agent dispatch ‚Äî the OPTIONAL-play graceful path, fully mechanical | autonomous |
| REC5 | F5 | `draft-reference-algorithms` still errors after 2 recovery attempts for one or more `interface_ids` | mechanical retry exhausted ‚Äî a human diagnoses the failing `interface_ids` | human |
| REC6 | F6 | `reference-algorithms.md` absent from `context/design/` after the skill reported execution | re-dispatch the skill to produce the missing file ‚Äî a mechanical re-run from inputs the builder already holds | autonomous |
| REC7 | F7 | prepare context package incomplete ‚Äî `plan.yaml` or `scenarios.yaml` missing alongside `tech.yaml` | warn about the incomplete package and continue using `tech.yaml` alone ‚Äî a mechanical warn-and-proceed | autonomous |

At Level 4, `intent-resolver` executes the autonomous entries (REC4, REC6, REC7) without a human; the human entries (REC1, REC2, REC3, REC5) always escalate.

