# submit-pr

Model-invocable skill for creating pull requests.

## Purpose

Push branch and create a pull request with the quality checklist embedded in the PR body. This skill executes — it does not orchestrate.

## Input

Receive from caller (agent):

| Field | Required | Description |
|-------|----------|-------------|
| `title` | Yes | PR title (conventional commit format) |
| `base` | Yes | Base branch (e.g., `main`) |
| `checklist` | Yes | Quality checklist from analyze-pr |
| `draft` | No | Create as draft PR (default: false) |
| `labels` | No | Labels to add |
| `assignees` | No | Assignees to add |

## Platform

This skill uses GitHub via `gh` CLI.

Other platforms (GitLab, Bitbucket) are not supported.

## Process

### 1. Push Branch

```bash
# Get current branch
branch=$(git branch --show-current)

# Check if branch has upstream, push if not
git rev-parse --abbrev-ref --symbolic-full-name @{u} 2>/dev/null || git push -u origin HEAD
```

### 2. Format PR Body

Transform the checklist into GitHub-flavored markdown:

```markdown
## Summary

{extract from commits or provide from input}

## Quality Checklist

### Required

- [ ] {must_have_item_1} — {trigger}
- [ ] {must_have_item_2} — {trigger}

### Optional

- [ ] {nice_to_have_item_1} — {trigger}
- [ ] {nice_to_have_item_2} — {trigger}

## Verification Evidence

| Item | Status | Evidence |
|------|--------|----------|
| {item} | {PASS/FAIL/REVIEW} | {evidence} |

---

🦅 Generated by [Phoenix OS]
```

### 3. Create PR

```bash
gh pr create \
  --title "{title}" \
  --body "{formatted_body}" \
  --base "{base}" \
  [--draft] \
  [--label "{label}"] \
  [--assignee "{assignee}"]
```

### 4. Get PR Info

```bash
gh pr view --json number,url,state,title
```

## Output

Produce output using template: `templates/pr-output.md`

**IMPORTANT**: This skill produces operation results. The calling agent receives this output and decides what to do next. Do NOT instruct the agent to return or stop.

## Constraints

- NEVER modify checklist content — only format it
- NEVER run tests or builds — just create the PR
- NEVER ask for user input — return result to caller
- ALWAYS use `gh pr create` (not API directly)
- ALWAYS include the "🦅 Generated by [Phoenix OS]" footer
