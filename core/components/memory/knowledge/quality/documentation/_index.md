# Documentation Quality Standards

QP-3 indexed checks for documentation completeness and developer-facing documentation quality. These checks assess whether documentation exists and is useful — not which documentation tools or formats to adopt (see `knowledge/` for tooling guidance).

## Files

- [api-architecture-docs.md](api-architecture-docs.md) — OpenAPI spec, ADR, architecture diagram presence and completeness | Patterns: OpenAPI, Swagger, ADR, C4, architecture diagram, API changelog
- [developer-onboarding.md](developer-onboarding.md) — README, CONTRIBUTING, environment setup, and onboarding completeness | Patterns: README, CONTRIBUTING, onboarding, env vars, local setup, troubleshooting
