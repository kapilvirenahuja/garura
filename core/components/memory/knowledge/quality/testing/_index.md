# Testing Quality Standards

Testing quality checks covering unit test structure, integration and E2E testing, coverage thresholds, and test pyramid compliance. These checks determine whether a test suite provides real confidence or only the appearance of it.

## Files

- [Unit Testing Standards](unit-testing.md) — AAA structure, mocking patterns, test isolation, assertion quality, mutation testing | Patterns: unit test, jest, pytest, arrange-act-assert, mocking, stubbing, mutation testing
- [Integration and E2E Testing Standards](integration-e2e-testing.md) — API tests, browser E2E, contract testing, DB integration tests | Patterns: integration test, e2e, cypress, playwright, pact, contract testing, supertest
- [Coverage and Test Patterns](coverage-test-patterns.md) — Coverage thresholds, test pyramid, mutation score, flaky test tracking | Patterns: coverage, test pyramid, mutation testing, flaky tests, branch coverage
