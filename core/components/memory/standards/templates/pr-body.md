# PR Body Template

Canonical shape for a pull request body. Every skill that creates or formats a PR body instantiates this template instead of inlining its own markdown.

Consumers: `submit-pr`, `analyze-pr`.

## Shape

```markdown
## Summary

{extract from commits or provide from input}

## Quality Checklist

### Required

- [ ] {must_have_item_1} — {trigger}
- [ ] {must_have_item_2} — {trigger}

### Optional

- [ ] {nice_to_have_item_1} — {trigger}
- [ ] {nice_to_have_item_2} — {trigger}

## Verification Evidence

| Item | Status | Evidence |
|------|--------|----------|
| {item} | {PASS/FAIL/REVIEW} | {evidence} |

## Eval Results

{eval_results if provided — formatted as a summary table with pass/fail counts and any failure details}

---

Generated by [Garura]
```

## Field Rules

| Field | Required | Rule |
|-------|----------|------|
| `summary` | yes | Extracted from commits or provided via input. Non-empty. |
| `checklist.required` | yes | At least one must-have item. Format: `- [ ] {item} — {trigger}`. |
| `checklist.optional` | no | Nice-to-have items. Same format as required. Omit section if empty. |
| `verification_evidence` | yes | Table with at least one row. Status values: `PASS`, `FAIL`, `REVIEW`. |
| `eval_results` | no | Conditional — see Eval Results rules below. |
| `footer` | yes | Always include `Generated by [Garura]` on its own line after `---`. |

## Eval Results Rules

- If `eval_results` is provided and all passed: include summary table in PR body
- If `eval_results` is provided with failures: include summary with failure details in PR body
- If `eval_results` is not provided: omit the Eval Results section entirely
