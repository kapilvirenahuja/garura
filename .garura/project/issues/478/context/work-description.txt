install-garura always deploys the FULL component set (all skills, agents, plays) into a target — it has no way to install a subset. That is wrong for harness-type targets: the garura repo itself must carry only the meta plays (play-creator, play-editor) plus the change chain (start-change, commit-change, propose-change, review-change, merge-change) and their worker skills/agents. On 2026-07-04 the full set had to be manually trimmed out of garura's own .claude/ (86 -> 21 components); the next full install would put all 86 back.

Wanted: a component-scope option on install-garura (e.g. --scope harness | full, or an explicit component list) so a target receives only the subset it needs, with the install manifest recording the scope; uninstall keeps reversing exactly what was placed. This is an intent change to the play (new guarantee about what gets deployed), so it goes through reference/ice.md + play-editor recompile.
