## Summary

**Problem:** `capture-learning` conflated extraction + LTM promotion in a single run, with no categorization of what type of learning was captured.

**Solution:** New `reap` play (L2 of the Learning Pipeline) compiled from hand-authored `intent.yaml` via `/create-play --build reap`. reap reads the build trinity (prepare → implement → validate), writes proposals to STM only (never LTM), and introduces a two-level learning taxonomy (`learning_category` + `sub_category`) aligned to `core/components/memory/`, with a structured invention escape hatch (`_proposed` flags + `taxonomy_justification`). `capture-learning` play deleted; supporting skills and agent retained and rewired.
