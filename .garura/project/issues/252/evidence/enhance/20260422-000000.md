## Enhancement Summary

**Problem:** Garura components (plays, agents, skills) had no validation layer. Structural errors, semantic cross-reference breaks, and ID collisions were only caught at runtime or PR review. The enhance pipeline's quality-check step passed trivially because no lint tooling existed.

**Solution:** Two-layer linting capability:
1. Deterministic Node.js CLI tool at `core/tools/lint-components/` — validates all Garura components across structural, semantic, and cross-reference rule tiers; emits structured JSON `{ violations, summary }`.
2. `lint-components` skill at `core/components/skills/lint-components/SKILL.md` — wraps the CLI, reads quality profile for severity classification, writes lint report artifact.
