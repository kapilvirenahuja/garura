# Fix-It Evidence: #195 — capture-learning extracts zero knowledge

## Issue
- **Number:** 195
- **Title:** fix: capture-learning extracts zero knowledge — only scans resolution-trace.yaml

## Root Cause
Step 3 of `core/components/recipes/capture-learning/SKILL.md` used a hardcoded `find -name "resolution-trace.yaml"` command. In real issues, `resolution-trace.yaml` is produced only by `prepare-implementation`'s LTM resolution protocol. Issues resolved via other recipes produce none of these files. Step 3 returned an empty list, and `knowledge-extractor` was invoked with zero paths.

Contributing cause: `knowledge-extractor.md` EXTRACT mode only extracted entries where `resolved_from == "llm"` — a field exclusive to `resolution-trace.yaml`. Other evidence types don't contain it.

## Fix Strategy
Broaden scan + extend extraction to multiple evidence types. Fixed at both layers:
1. Updated `intent.yaml` C12 to enumerate 7 recognized artifact types; updated C15 for broader zero-candidate condition
2. Extended `knowledge-extractor.md` EXTRACT mode with 6 per-format extraction heuristics
3. Rebaked `SKILL.md` from updated intent — Step 3 now scans all recognized artifact types
4. Synced to global deployment
